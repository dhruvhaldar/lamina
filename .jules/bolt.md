## 2026-04-06 - [ABD Matrix Assembly and Inversion Optimization]
**Learning:** In `Laminate.update`, assembling the 6x6 ABD matrix using `np.vstack` and `np.hstack` adds significant array allocation overhead compared to pre-allocating an `np.empty((6, 6))` array and filling it via slicing. Furthermore, immediately computing the compliance matrix (`abd`) via `np.linalg.inv(self.ABD)` is unnecessary and computationally expensive for laminates that are discarded early (e.g., failing buckling constraints) and do not require full failure analysis.
**Action:** When assembling small block matrices, allocate an empty array and use slicing for a 3-4x speedup. Implement lazy evaluation (using `@property` and a hidden cache attribute) for expensive matrix inversions so they are only computed on-demand.
