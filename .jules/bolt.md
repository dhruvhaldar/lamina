## 2024-05-17 - Precomputing Laminate Invariants for Optimization Loops
**Learning:** In optimization loops (`calculate_safety_factor`), redundant element-wise array calculations (e.g., squaring trig functions or finding geometric midpoints) add significant overhead when executed thousands of times.
**Action:** When working on object-oriented mathematical structures evaluated inside hot loops (e.g., within genetic algorithms), cache frequently used invariant arrays (such as geometric midpoints or squared trigonometric arrays like `c * c`) directly on the object during initialization. Reusing these cached attributes via `hasattr` checks avoids repetitive, costly element-wise NumPy array calculations and allocations in downstream hot-path functions.
