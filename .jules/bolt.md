## 2026-04-06 - [ABD Matrix Assembly and Inversion Optimization]
**Learning:** In `Laminate.update`, assembling the 6x6 ABD matrix using `np.vstack` and `np.hstack` adds significant array allocation overhead compared to pre-allocating an `np.empty((6, 6))` array and filling it via slicing. Furthermore, immediately computing the compliance matrix (`abd`) via `np.linalg.inv(self.ABD)` is unnecessary and computationally expensive for laminates that are discarded early (e.g., failing buckling constraints) and do not require full failure analysis.
**Action:** When assembling small block matrices, allocate an empty array and use slicing for a 3-4x speedup. Implement lazy evaluation (using `@property` and a hidden cache attribute) for expensive matrix inversions so they are only computed on-demand.

## 2026-03-09 - [NumPy Overhead & Algebraic Simplifications in Tight Loops]
**Learning:** Functions like `np.linspace` or computing exponents (`zk**3`) using Numpy abstractions introduce unnecessary overhead for smaller datasets in tight loops (e.g. `Laminate.update` calling `_calculate_z_coords`). Using simpler alternatives like `np.arange` with native math operations, and factorizing array arithmetic `(zk**2 - zk_1**2)` to `(zk - zk_1) * (zk + zk_1)` drastically reduces execution time. Also, keeping intermediate mathematical terms (like pre-computing `U2 * cos2` to reuse) directly in the local namespace rather than making array allocations helps minimize repetitive matrix processing times.
**Action:** When working with repetitive inner loops over small numpy arrays, rely on algebraic factorizations over default vector exponentiation rules. Also, avoid higher level numpy features like `np.linspace` and use simpler math against initialized `np.arange` elements instead.
