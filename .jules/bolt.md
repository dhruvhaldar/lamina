## 2026-03-10 - [Avoid Transpose and Reshape in Tight Vectorized Loops]
**Learning:** In highly vectorized numerical code (like coordinate transformation and stiffness matrix generation in `lamina/clt.py`), using standard NumPy idioms like `np.array([...]).transpose()` or creating a multidimensional array `(3, 3, N)` just to immediately call `.reshape(9, -1)` introduces surprising memory allocation overhead. Native slicing into pre-allocated `np.empty()` arrays (e.g. `res[0] = Q_bar_11` for a `(9, N)` array instead of `(3, 3, N)`) bypasses intermediate object creation and results in ~15-20% speedups. Similarly, relying on `np.divide` directly with a context manager `with np.errstate(divide='ignore'):` is up to 20% faster than boolean array masking (`mask = a != 0; res[mask] = 1 / a[mask]`).
**Action:** When a method generates a multi-dimensional array that the caller immediately reshapes or flattens, refactor the generator to return the flat shape directly via `np.empty()`. Replace boolean masks guarding division with `np.errstate` when the array size justifies skipping the double-pass of mask creation and masked assignment.

## 2026-04-06 - [Vectorized Optimizations using errstate and np.where]
**Learning:** In highly vectorized operations with small arrays (like solving the Tsai-Wu criterion for laminates), explicitly calculating masks and instantiating empty/full subset arrays based on valid ranges causes significant object allocation overhead inside loops. By calculating potential linear and quadratic solutions globally leveraging `np.errstate(divide='ignore', invalid='ignore')`, we completely bypass Python array loops and boolean masking sub-assignments. We also found out replacing broadcast mechanisms like `eps0[:, np.newaxis] + kap[:, np.newaxis] * z[np.newaxis, :]` with simple vector algebra like `ex = eps0[0] + kap[0] * z` avoids costly intermediary 2D/3D array construction. These micro-optimizations combined yielded around 20-30% faster execution.
**Action:** Use `np.errstate` along with `np.where` for branchless logic in vectorized code rather than manually masking valid values, and prefer native Python math / component-based logic over creating large broadcasted axes for small dimension arithmetic.

## 2026-04-06 - [Avoid np.linspace for Small Arrays in Hot Paths]
**Learning:** Using `np.linspace(0, 2*np.pi, num_points)` in hot paths like failure envelope generation (`FailureCriterion` methods) introduces measurable function abstraction overhead when calculating stresses over a small number of points. Replacing it with `np.arange(num_points, dtype=np.float64) * (2 * np.pi / (num_points - 1))` is approximately 3-4x faster because it bypasses `np.linspace`'s internal validation, broadcasting, and endpoint handling, directly executing simpler native array math.
**Action:** For small, fixed-size numerical array generation in performance-critical loops or functions that run frequently, prefer `np.arange` combined with simple algebraic scaling over higher-level abstractions like `np.linspace`.
