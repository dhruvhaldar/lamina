## 2024-05-18 - [Defense in Depth for Path Traversal]
**Vulnerability:** The static file serving endpoint (`/{filename}`) relied exclusively on `os.path.commonpath` to detect path traversal attempts after joining paths. While technically secure, this misses an opportunity for early rejection and risks edge cases in standard library resolution logic.
**Learning:** Relying solely on `os.path.commonpath` means the input string containing directory traversal characters (`/`, `\`) is still processed by path manipulation functions (`os.path.join`, `os.path.realpath`). This represents a missed opportunity for defense-in-depth and early rejection at the API boundary, which is a stronger security posture.
**Prevention:** In static file serving endpoints, implement defense-in-depth by explicitly rejecting directory traversal characters (`/` and `\`) at the API boundary (e.g., returning 400 Bad Request) before passing them to OS path resolution mechanisms.
