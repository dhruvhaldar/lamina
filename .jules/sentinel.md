## 2026-03-22 - [Defense-in-Depth Path Traversal Prevention]
**Vulnerability:** The static file serving endpoint `read_file` relied exclusively on `os.path.commonpath` to verify that requested files were securely contained within the `/public` directory. While technically effective against basic traversal, URL-encoded path separators (`%2F`) could still reach OS-level path parsing, increasing the attack surface if underlying OS path handling had edge-case behaviors.
**Learning:** Security controls should fail early and explicitly reject known bad input (like directory separators in a flat-file serving endpoint) rather than relying solely on secondary verifications (like OS path boundary checking) later in the execution flow.
**Prevention:** Always implement strict, explicit input validation at the API boundary. Reject filenames containing directory traversal characters (`/` or `\\`) before passing user-controlled strings to OS filesystem APIs to establish a robust defense-in-depth architecture.
